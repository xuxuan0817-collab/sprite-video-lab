# Sprite Video Lab 详细使用说明

语言：[English](./USAGE.md) | [简体中文](./USAGE.zh-CN.md)

这份说明面向日常使用者，按实际操作顺序讲清楚从导入素材到导出 Sprite 的完整流程。项目本身是本地工具：素材、临时文件、导出结果都保存在你的电脑上，不需要上传到外部服务。

## 适合处理什么素材

Sprite Video Lab 适合把视频片段或单张图片整理成 2D 游戏素材，尤其适合这些场景：

- 绿幕、蓝幕、纯色背景角色动作。
- AI 生成的视频角色、特效或建筑素材。
- 火焰、闪电、发光、粒子等需要保留亮部透明感的 VFX。
- 横向很长的攻击轨迹、刀光、冲刺残影、多姿态长条图。
- 已经抠好或不需要抠图，只想统一尺寸、对齐、抽帧和打包的 PNG 序列。

支持导入常见视频和图片格式：

- 视频：`mp4`、`mov`、`mkv`、`webm`
- 图片：`png`、`jpg`、`jpeg`、`webp`、`bmp`

## 安装和启动

### 1. 安装基础环境

准备 Python 3.10 或更高版本，然后在项目目录里执行：

```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

基础功能只需要 `requirements.txt`。如果只处理纯色背景、抽帧、缩放、导出，不需要安装 AI 依赖。

### 2. 安装 ffmpeg

视频导入和抽帧需要 `ffmpeg` 和 `ffprobe`。推荐把它们加入 `PATH`。

如果你把 ffmpeg 放在独立目录，可以在启动前设置：

```powershell
$env:SPRITE_VIDEO_LAB_FFMPEG_DIR="D:\ffmpeg\bin"
```

### 3. 可选：安装 AI 抠图环境

如果要使用 BiRefNet 或 CorridorKey，运行：

```bat
setup_ai_runtime.bat
```

这个脚本会安装 AI 依赖，并在可用时准备 CorridorKey。第一次使用 AI 模型时会下载模型文件，时间取决于网络和模型大小。

常用环境变量：

```bat
set SPRITE_VIDEO_LAB_AI_MODEL_CACHE=E:\sprite-video-lab-models\huggingface
set SPRITE_VIDEO_LAB_CORRIDORKEY_ROOT=E:\sprite-video-lab-models\CorridorKey
set SPRITE_VIDEO_LAB_PYTHON=C:\path\to\python.exe
```

更多 AI 说明见 [AI_MATTING.md](./AI_MATTING.md)。

### 4. 启动工具

Windows 推荐直接双击或执行：

```bat
start_sprite_video_lab.bat
```

也可以手动启动：

```powershell
python server.py
```

默认打开地址：

```text
http://127.0.0.1:8894
```

需要换端口时：

```powershell
python server.py --host 127.0.0.1 --port 8895
```

## 四步工作流

界面按 4 个步骤组织：导入素材、预览与截取区间、抽帧与抠图、检查帧并导出。

### 第 1 步：导入素材

你有两种方式导入：

- 在“本地视频/图片路径”里输入完整文件路径，然后点“导入路径”。
- 把文件拖到上传区域，或点击上传区域选择文件。

导入成功后，界面会显示素材名称、分辨率、帧率、时长等信息。图片没有视频时长，工具会把它当作单帧素材处理。

建议：

- 路径里有中文也可以使用，但尽量用复制出来的完整路径。
- 大视频可以先用剪辑软件粗剪，只保留需要处理的片段，能明显减少等待时间。
- 如果是透明 PNG，通常可以关闭抠背景，只做尺寸归一和导出。

### 第 2 步：预览与截取区间

视频素材会显示预览播放器。你可以拖动预览进度，也可以直接调整“起始帧”和“结束帧”。

区间设置会实时生效，不需要额外确认。第 3 步处理时会直接使用当前选区。

使用建议：

- 起始帧尽量放在动作刚开始前 1-2 帧，避免丢掉第一帧动作。
- 结束帧放在动作完全收尾后，后面多余静止帧可以在第 4 步取消勾选。
- 如果动作很短，先把“保留每 N 帧”设为 1，避免抽得太稀。

## 第 3 步：抽帧与抠图

这一部分决定输出帧的数量、尺寸、对齐方式和抠图质量。建议先点“预览当前帧效果”，确认参数，再点“开始处理”。

### 保留每 N 帧

控制抽帧密度：

- `1`：保留每一帧，最顺滑，数量最多。
- `2`：每 2 帧保留 1 帧，常见默认选择。
- `3` 或更高：适合原视频帧率很高、动作较慢的素材。

如果导出后动画太快或太跳，可以回到这里降低数值。

### 目标高度

输出素材会按目标高度等比缩放。比如设置为 `128`，每一帧主体高度会接近 128 px。

建议：

- 像素风小角色：`64`、`96`、`128`
- 中型角色动作：`192`、`256`
- 大型 VFX 或高清素材：`512` 以上

目标高度越大，AI 抠图和导出越慢，文件也会更大。

### 画布布局

画布决定每一帧 PNG 的尺寸和主体摆放方式。

- `自动宽度居中`：适合横向攻击、长条 VFX、宽动作。工具会根据内容自动保留足够宽度。
- `正方形底部对齐`：适合角色站立、行走、跳跃等有地面基准的动作。
- `正方形居中`：适合图标、魔法球、爆炸、没有明确地面的素材。

如果角色在动画预览里左右跳动，通常需要增加画布边距，或换成正方形布局。

### 画布边距

控制主体四周额外留白。边距太小会裁切动作边缘；边距太大会浪费 Sprite Sheet 空间。

常用范围：

- 普通角色：`8` 到 `16`
- 攻击动作、武器挥舞：`16` 到 `32`
- 大发光、粒子、爆炸：`32` 以上

## 抠图模式怎么选

### No matting

不抠图，只缩放、对齐、导出。适合已经带透明通道的 PNG，或你只想做抽帧和打包。

### Solid color / green screen

纯色背景抠图，速度最快，适合绿幕、蓝幕、白底、黑底等颜色明确的素材。

常用参数：

- 抠图取色方式：自动取角落，或手动指定背景色。
- 抠图阈值：越大，越容易把接近背景色的像素删掉。
- 软边过渡：越大，边缘越柔和。
- 去色溢出强度：减少绿色或蓝色边缘污染。
- Halo 收缩像素：收紧 alpha 边缘，用来处理脏边。

调参顺序建议：

1. 先确定背景色取对了。
2. 调阈值，让背景大体消失。
3. 调软边，让边缘自然。
4. 加去色溢出和 Halo，处理绿边、蓝边、白边。

### BiRefNet

AI 主体抠图，适合背景不纯、AI 生成背景、真实环境背景。它会根据主体语义生成 alpha。

适合：

- 人物、怪物、建筑、道具等有明确主体的素材。
- 背景不是单一颜色，纯色抠图难以处理的素材。

不太适合：

- 大量透明发光粒子。
- 主体和背景语义混在一起的画面。
- 需要严格保留所有火焰、闪电尾迹的素材。

### BiRefNet + Luma

AI 主体 alpha 加亮度 alpha。它会在保留主体的同时，尽量保留亮部发光、火焰、闪电、粒子。

适合：

- VFX 素材。
- 魔法、火焰、爆炸、闪电。
- AI 生成的发光角色或建筑光效。

主要参数：

- Luma 黑场：低于这个亮度的区域更容易透明。
- Luma 白场：高于这个亮度的区域更容易保留。
- Luma Gamma：改变亮度过渡曲线。
- Luma 强度：控制亮度 alpha 对最终结果的影响。

调参方向：

- 亮部丢失：降低 Luma 白场，或提高 Luma 强度。
- 暗部残留太多：提高 Luma 黑场。
- 发光边缘太硬：适当提高 Gamma 或软边。
- 主体内部被抠透明：使用“主体保护预设”。

## 主体保护预设

主体保护预设是给 `BiRefNet + Luma` 准备的快捷按钮，用来避免建筑、角色内部被亮度逻辑抠成半透明。

三个预设：

- `温和`：保守增强，适合主体已经比较完整，只是略有透明。
- `中等`：常用选择，兼顾主体完整和发光保留。
- `强力`：尽量保主体，适合建筑内部、盔甲、暗部结构被抠坏的情况。

点击预设后会自动：

- 开启抠背景。
- 切换到 `BiRefNet + Luma`。
- 关闭 CorridorKey。
- 把 Halo 设置为 `0`。
- 把 AI 边缘精细度设为较高值。
- 写入一组 Luma 参数。

使用方式：

1. 先选择当前帧里最容易出问题的一帧。
2. 点“预览当前帧效果”。
3. 如果主体内部变透明，点一个主体保护预设。
4. 再预览一次，比较结果。
5. 确认后再点“开始处理”。

## CorridorKey 精修

CorridorKey 用于绿幕或蓝幕素材的边缘重建。它不是通用 AI 抠图，而是更适合色幕素材的前景颜色和 alpha 修复。

适合：

- 真实绿幕或蓝幕。
- alpha 大体正确，但边缘有绿色或蓝色污染。
- 半透明边缘、头发、柔光边需要更自然。

不建议用于：

- 非绿幕/蓝幕背景。
- 发光粒子为主的素材。
- 已经使用 `BiRefNet + Luma` 处理 VFX 的素材。

如果素材很大，工具会自动使用更安全的大图处理路径，降低 GPU 后处理阶段的风险。

## AI 抠图功能完整使用指南

这一节是给实际处理素材时看的。你不需要理解模型细节，只需要先判断素材类型，再按对应流程调参数。

### 先选哪个模式

| 你的素材长什么样 | 推荐模式 | 不推荐 |
| --- | --- | --- |
| 纯绿幕、纯蓝幕、纯白底、纯黑底 | `Solid color / green screen` | 一开始就用 BiRefNet，容易慢而且边缘不一定更干净 |
| 真实背景、AI 背景、背景颜色不统一，但主体很明确 | `BiRefNet` | CorridorKey，它只适合绿幕/蓝幕 |
| 火焰、闪电、爆炸、发光粒子、魔法光效 | `BiRefNet + Luma` | 纯 BiRefNet，可能把亮部尾迹当背景删掉 |
| 建筑、角色、道具本体被抠成半透明 | `BiRefNet + Luma` + 主体保护预设 | 提高 Halo，可能让主体更破 |
| 绿幕/蓝幕边缘有明显绿色或蓝色污染 | `Solid color` 或 `BiRefNet` 之后开启 CorridorKey | 用 Luma 解决色幕污染，通常方向不对 |
| 已经是透明 PNG | `No matting` | 重新 AI 抠图，可能破坏原 alpha |

### BiRefNet 怎么用

BiRefNet 是“找主体”的 AI 抠图。它适合主体明确、背景复杂的素材，比如 AI 生成角色、怪物、建筑、道具、头像、立绘。

操作步骤：

1. 在第 3 步打开“启用抠背景”。
2. 把“抠图模式”选为 `BiRefNet`。
3. “BiRefNet 模型”优先选 `BiRefNet HR-matting`，显存或速度吃紧时再选 `lite-2K`。
4. “AI 运行方式”保持 `auto`；如果 GPU 有问题，再改成 `cpu`。
5. “AI 边缘精细度”先用 `1024`，边缘细节不够再试 `1536`、`2048`、`2560`。
6. 找一帧最难处理的画面，点“预览当前帧效果”。
7. 如果主体完整、边缘可接受，再点“开始处理”。

BiRefNet 参数怎么调：

| 现象 | 调整方式 |
| --- | --- |
| 边缘太糊、头发或轮廓丢细节 | 提高“AI 边缘精细度”，比如从 `1024` 到 `2048` |
| 处理太慢或显存不足 | 降低“AI 边缘精细度”，或换 `lite-2K`，或把运行方式改成 `cpu` |
| 主体内部出现透明洞 | 改用 `BiRefNet + Luma`，再点主体保护预设 |
| 发光、火焰、闪电尾迹被删掉 | 改用 `BiRefNet + Luma` |
| 背景残留一大片 | 先确认选区帧对不对；如果主体和背景混在一起，换一帧预览或改用纯色抠图 |

### BiRefNet + Luma 怎么用

`BiRefNet + Luma` 是“主体抠图 + 亮度保留”。它最适合 VFX：火焰、闪电、爆炸、光圈、粒子、魔法轨迹。普通 BiRefNet 容易把这些亮部当成背景删掉，Luma 会把亮的部分补回来。

操作步骤：

1. 在“抠图模式”里选择 `BiRefNet + Luma`。
2. 先把 Halo 设为 `0`，避免发光边缘被收缩掉。
3. CorridorKey 先关闭，除非你处理的是标准绿幕/蓝幕边缘。
4. “AI 边缘精细度”先设 `1024` 或 `1536`；高清 VFX 可以试 `2048`。
5. 点“预览当前帧效果”。
6. 根据预览调 Luma 黑场、白场、Gamma、强度。
7. 预览通过后再点“开始处理”。

Luma 四个参数可以这样理解：

| 参数 | 它控制什么 | 往上调会怎样 | 往下调会怎样 |
| --- | --- | --- | --- |
| Luma 黑场 | 多暗的像素开始被当作背景 | 暗部残留更少，但可能吃掉暗色细节 | 保留更多暗部，但背景脏点可能变多 |
| Luma 白场 | 多亮的像素一定保留 | 只有更亮的光效会保留，结果更干净 | 更多光效和浅色区域会保留 |
| Luma Gamma | 亮度过渡曲线 | 过渡更柔或更偏保留中间亮度，适合柔光 | 过渡更硬，适合干净电光和高对比素材 |
| Luma 强度 | Luma 对最终 alpha 的影响 | 光效更完整，但背景亮点也可能留下 | 更依赖 BiRefNet 主体 alpha，背景更干净 |

常用起点：

- 轻微发光角色：白场 `110`，Gamma `0.7`，强度 `1.3`
- 普通魔法/火焰：白场 `85`，Gamma `0.55`，强度 `1.7`
- 强闪电/爆炸：白场 `65`，Gamma `0.45`，强度 `2.0`

典型问题：

| 现象 | 调整方式 |
| --- | --- |
| 火焰、闪电不够完整 | 降低 Luma 白场，或提高 Luma 强度 |
| 背景亮点太多 | 提高 Luma 黑场，或降低 Luma 强度 |
| 主体内部变透明 | 点“主体保护预设”的 `中等` 或 `强力` |
| 发光边缘被切硬 | Halo 设为 `0`，适当提高软边，调整 Gamma |
| 整体像一团亮雾 | 提高 Luma 黑场，降低 Luma 强度，必要时改回 BiRefNet |

### 主体保护预设怎么选

主体保护预设是专门给 `BiRefNet + Luma` 准备的。它解决的问题不是“抠不干净”，而是“主体内部被亮度规则误伤，变得半透明”。

| 预设 | 什么时候用 |
| --- | --- |
| 温和 | 主体基本完整，只是局部有一点透明 |
| 中等 | 默认优先尝试，适合大多数建筑、角色、道具 |
| 强力 | 主体内部大面积透明，或建筑结构、盔甲、暗部纹理被吃掉 |

推荐流程：

1. 先用 `BiRefNet + Luma` 预览。
2. 如果光效保住了，但主体变透明，点 `中等`。
3. 再预览；如果还透明，点 `强力`。
4. 如果主体保住了但背景也脏，略微提高 Luma 黑场或降低 Luma 强度。

### CorridorKey 怎么用

CorridorKey 是色幕精修，不是通用抠图。它适合绿幕/蓝幕素材的边缘重建，尤其是边缘有绿色/蓝色溢出、半透明边、头发、柔光边的情况。

使用前先确认：

- 素材背景真的是绿幕或蓝幕。
- 普通抠图或 BiRefNet 已经能得到大体正确的 alpha。
- 主要问题是边缘颜色脏，不是主体识别失败。

操作步骤：

1. 先用 `Solid color / green screen` 或 `BiRefNet` 做出一个大体正确的预览。
2. 打开 `CorridorKey 精修`。
3. “色幕类型”保持 `Auto`，如果自动判断错了，再手动选 `Green` 或 `Blue`。
4. 保持去色溢出强度在 `0.8` 到 `1.5` 之间。
5. 点“预览当前帧效果”，放大看边缘。
6. 如果边缘变干净，再开始整段处理。

不要在这些情况下用 CorridorKey：

- 背景不是绿幕/蓝幕。
- 素材主要是火焰、闪电、粒子、光效。
- 你已经在用 `BiRefNet + Luma` 保留 VFX。
- 问题是主体识别错了，而不是色幕边缘污染。

CorridorKey 常见问题：

| 现象 | 调整方式 |
| --- | --- |
| 边缘仍有绿/蓝 | 手动选色幕类型，提高去色溢出，必要时加 Halo `1` |
| 边缘被吃掉 | 降低抠图阈值或 Halo，先让基础 alpha 更完整 |
| 处理慢 | 降低目标高度或 AI 边缘精细度，先用单帧预览确认 |
| 效果比不开更差 | 关闭 CorridorKey，改用普通去色溢出和后处理 |

### 推荐的完整工作流

处理一批素材时，不要直接点“开始处理”。更稳的做法是：

1. 导入素材并截取区间。
2. 在动作最复杂的一帧停住。
3. 根据素材类型选择抠图模式。
4. 点“预览当前帧效果”。
5. 放大查看边缘、主体内部、发光区域。
6. 根据问题调 BiRefNet / Luma / CorridorKey。
7. 必要时用“残绿涂黑”或“半透涂黑”测试边缘。
8. 确认单帧结果后，再决定是否开启批处理后处理。
9. 点“开始处理”。
10. 在第 4 步用动画预览检查连续性，再导出。

## 单帧预览和预览后处理

“预览当前帧效果”会只处理当前帧，方便你快速调参数。预览区会显示原图和处理结果，并支持缩放查看边缘。

预览后可以直接做三件事：

- `残绿涂黑`：把预览图里残留的绿色像素改成黑色，但保留原 alpha。
- `半透涂黑`：把半透明像素的 RGB 改成黑色，但保留 alpha。
- `下载预览图`：直接下载当前处理结果 PNG。

这些按钮只作用于当前预览图，不会自动影响整段处理。如果希望整段都应用，需要开启“批处理后处理”里的对应选项。

## 批处理后处理

批处理后处理会在点“开始处理”时应用到每一帧输出图。

选项：

- `残绿涂黑`：适合边缘仍有绿色污染，但你准备用黑底、深色描边或后续引擎混合。
- `半透涂黑（保留 alpha）`：适合半透明边缘 RGB 太亮，导入游戏后出现白边或彩边。

注意：

- 这两个选项都会改变像素颜色，但不会删除 alpha。
- 如果素材需要在浅色背景上显示，先用单帧预览确认不会产生黑边。
- 对发光素材谨慎使用“半透涂黑”，否则可能削弱彩色光晕。

## 第 4 步：检查帧并导出

处理完成后会进入结果区。这里可以检查帧、选择需要导出的帧、预览动画，并生成最终文件。

### 帧选择

可用按钮：

- `全选`
- `全不选`
- `奇数帧`
- `偶数帧`
- `反选`

你也可以手动勾选或取消某一帧。

常见用法：

- 动作太慢：只保留奇数帧或偶数帧。
- 开头或结尾有空帧：手动取消。
- 循环不顺：取消最后一帧或第一帧，避免重复停顿。

### 动画预览

动画预览会循环播放当前已选帧。

可以调整：

- 预览背景色：检查白边、黑边、透明边缘。
- 帧间隔：模拟游戏里的播放速度。
- 倒序播放 / 导出：让动画按反方向播放，并按倒序导出。

“倒序播放 / 导出”会影响最终导出顺序。比如你想把右挥改成反向收招，可以开启后再导出。

### 导出

设置 Sprite Sheet 列数后，点击“导出选中帧”。

导出结果包括：

- `frames/`：透明 PNG 帧序列。
- `sprite_sheet.png`：合成后的 Sprite Sheet。
- `frames.zip`：打包好的帧文件。
- `export.json`：导出清单，包含帧尺寸、顺序、sheet 信息等。

如果开启倒序导出，`frames/`、`sprite_sheet.png` 和 `export.json` 的帧顺序都会按当前倒序选择处理。

## 常见参数组合

### 绿幕角色动作

- 抠图模式：`Solid color / green screen`
- 取色方式：手动背景色，选真实绿幕色
- 阈值：`70` 到 `100`
- 软边：`20` 到 `40`
- 去色溢出：`0.8` 到 `1.5`
- Halo：`1` 或 `2`
- 如果边缘仍脏：开启 CorridorKey 精修

### AI 生成角色或建筑

- 抠图模式：`BiRefNet`
- AI 模型：`BiRefNet HR-matting`
- AI 运行方式：`auto`
- AI 边缘精细度：`1024` 到 `2048`
- 如果内部被抠透明：改用 `BiRefNet + Luma` 并点主体保护预设

### 火焰、闪电、粒子

- 抠图模式：`BiRefNet + Luma`
- Luma 白场：降低到 `65` 到 `120`
- Luma 强度：`1.3` 到 `2.0`
- Halo：`0`
- CorridorKey：关闭
- 批处理后处理：通常先不要开，除非确认需要去除残绿或半透明亮边

### 横向攻击轨迹

- 画布布局：`自动宽度居中`
- 画布边距：`24` 到 `48`
- 保留每 N 帧：`1` 或 `2`
- 预览时检查左右边缘有没有被裁切

### 已有透明 PNG

- 抠背景：关闭，或模式选择 `No matting`
- 画布布局：按素材类型选择
- 目标高度：设置成项目需要的像素尺寸
- 直接处理并导出

## 故障排查

### 打不开视频或提示 ffmpeg 不可用

- 确认 `ffmpeg.exe` 和 `ffprobe.exe` 在 `PATH` 中。
- 或设置 `SPRITE_VIDEO_LAB_FFMPEG_DIR` 指向它们所在目录。
- 重启终端后再启动工具。

### AI 抠图第一次很慢

第一次会下载模型并加载到内存，属于正常情况。后续会复用缓存。

### CUDA 不可用

把 AI 运行方式设为 `cpu`，或者保持 `auto` 让工具自动选择。CPU 能运行，但会慢很多。

### 边缘有绿边或蓝边

优先尝试：

1. 手动选择真实背景色。
2. 提高去色溢出强度。
3. 设置 Halo 为 `1` 或 `2`。
4. 绿幕/蓝幕素材开启 CorridorKey。
5. 仍有残留时，用预览或批处理的“残绿涂黑”。

### 导入游戏后出现白边

很多白边来自半透明像素的 RGB 颜色。可以尝试“半透涂黑（保留 alpha）”，但一定先在浅色和深色背景上预览。

### Sprite Sheet 太大

- 降低目标高度。
- 增大“保留每 N 帧”。
- 减少画布边距。
- 取消不需要的帧。
- 增加 Sprite Sheet 列数只会改变排布，不会减少总像素量。

### 动画方向反了

在第 4 步开启“倒序播放 / 导出”，确认预览后重新导出。

## 实用工作习惯

- 每次先用单帧预览调参数，不要直接处理长片段。
- 选动作中最复杂、边缘最脏、光效最多的一帧做预览。
- 导出前用不同背景色检查边缘。
- 对同一批素材保持目标高度、画布布局和边距一致，方便游戏里对齐。
- 重要素材保留 `export.json`，方便以后追踪导出参数和帧顺序。
